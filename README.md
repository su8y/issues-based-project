# Github 기반 프로젝트를 위한 프로젝트 템플릿

이 저장소는 Github Issue 기반 프로젝트를 성공적으로 완수하기 위한 가이드 라인을 제공합니다.

[기여(참여)가이드](contributing.md)

## 왜 Github-Issue 기반인 것인가 ?
Issue는 단순히 할 일이 아니라 왜 이 작업이 필요한지에 관한 **공식 기록**이다.
- 문제의 배경
- 기대 동작
- 논의 히스토리
- 설계 의도
즉, 코드만 봐서는 알수없는 맥락이 포함된다.

### 다수의 협업시 장점 제공
issue 기반으로 작업을 한다면 **Assignee**가 명시가 되어 진행 상태가 **공유**가 되고 중복작업이 차단이 된다.
issue 기반이 아니라면 아래와 같은 단점을 가질수 있다.
- 같은 내용을 여러 사용자가 수정
- PR이 겹치거나 충돌
- "이거 누가 하고있었나요?" 발생

### PR 리뷰 품질이 압도적으로 좋아진다
issue가 없는 PR은 "왜 이걸 바꿨는지", "요구사항이 무엇이었는지"를 알기 위해 많은 노력(리소스)가 필요하다.

issue 기반 PR은 변경 이유가 issue에 있고 reviewer는 구현에만 집중해서 리뷰를 할 수 있다.
```
Fixed: #123
```
