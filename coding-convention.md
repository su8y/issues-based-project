# 코딩 규칙

프로젝트는 코딩 규칙을 PR 단위의 CI(Continuous Integration) 를 통해 검증한다.
다만, 코딩 스타일과 세부 규칙은 자율을 원칙으로 운영한다.

자율로 정하는 이유
- 프로젝트와 언어 프레임워크 별로 권장하는 코딩 규칙이 다름.
- 팀 별로 이미 정해진 코딩 규칙이 있기 때문에 추가 학습 곡선을 추가하지 않으려는 목적

### CI의 역할
CI는 코딩 스타일을 강제하기 위한 도구가 아니라, 다음 항목을 자동으로 보장하기 위한 최소한의 안전장치이다.

- 빌드 가능 여부
- 테스트 통과 여부 (기존의 동작을 변경하지 않는가 ?)
- 포맷/정적 분석 오류 탐지
